# Learning: API to Agent Wiring & Asynchronous Export Strategies

## Context
During MVP 2 (JD-46 and JD-47), the Cover Letter and Question Answer APIs were stubbed out (returning 404/503 errors). This step involved wiring them directly to the `CoverLetterAgent` and `QAAgent` components that were previously implemented.

## Key Decisions

### Asynchronous Execution (Background Tasks vs. Temporal)
- The initial design proposed using Temporal for the Cover Letter agent. However, for a single synchronous LLM generation task (which typically resolves in ~5–15 seconds and does not have complex multi-step state orchestration like scraping), FastAPI's `BackgroundTasks` provide a lightweight, decoupled mechanism. 
- We updated `POST /api/v1/cover-letter/{job_id}` to return a `202 Accepted` immediately with status `pending`, while the background task handles LLM generation and database updates asynchronously.

### Binary Formats and Export Endpoints
- Requirements stated the need to export Cover Letters as PDF or Markdown.
- We added `weasyprint` and `markdown` as dependencies. The workflow natively transforms the Markdown string generated by the LLM into HTML (`markdown.markdown(cl.content)`) and feeds it directly into `weasyprint.HTML` to generate a byte stream, maintaining serverless capabilities without heavy PDF libraries.

## Lessons Learned
1. **API Stubs Avoidance:** Empty endpoints or returning 503 should be replaced as early as possible once the underlying domain/agent logic is stable.
2. **Context Passing in RAG MVP:** Passing structured context extracted from the `Job` object directly into the QA LLM prompt allows for rapid prototyping of RAG workflows before a fully optimized `pgvector` index is needed.
